SDD is a structured workflow that forces AI agents to think before coding. Instead of generating code directly from a prompt, the AI follows a deterministic pipeline: understand intent → document requirements → design solution → implement → verify.

Key Principle

No Artifact Skipping

The SDD pipeline mandates that no phase can skip artifacts. If Apply produces code that doesn't match Spec, Verify catches it. This creates an audit trail and reduces rework, eliminating the "generate-and-pray" pattern.

Phase Logic Integration

Each phase produces an artifact that feeds the next. Skills define specific rules, while Engram persists state across sessions for pick-up-and-go development.

Self-Healing Cycles

Every failure in the Verify phase automatically triggers a task re-decomposition, ensuring the system remains self-correcting and high-performance.
