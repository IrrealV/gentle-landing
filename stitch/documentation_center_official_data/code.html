02

Engram Persistence

Engram stores observations in a local SQLite database with full-text search (FTS5). It survives session restarts, context compaction, and agent switches.

Core Operations

  • mem_save
    Store Observation

    Used after decisions, bugfixes, or discoveries.

  • mem_search
    FTS5 Query

    Retrieve historical context via semantic search.

  • mem_context
    Recent History

    Load relevant history at session start.

  • mem_session_summary
    Close Session

    Synthesize achievements before ending work.

engram_schema.sql 
CREATE TABLE observations (
  id INTEGER PRIMARY KEY,
  type TEXT, -- decision|bugfix|arch
  title TEXT,
  content TEXT,
  project TEXT,
  topic_key TEXT, -- for upserts
  created_at DATETIME
);

CREATE VIRTUAL TABLE obs_fts 
USING fts5(content, tokenize='porter');
